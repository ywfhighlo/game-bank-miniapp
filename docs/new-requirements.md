# 项目需求文档

## 项目目标
本项目旨在开发一个微信小程序，通过将运动时间兑换为游戏时间的方式，鼓励用户（特别是青少年）培养健康的运动习惯。

## 功能需求
# 新功能需求规划

## 增加倒计时和日历工具

1. 倒计时功能
功能目标
在首页显示寒暑假倒计时，激励学生学习和运动，让他们对假期充满期待。
具体需求
显示位置
在首页菜单栏增加"假期倒计时"选项
与"运动记录"、"开始游戏"并列显示
倒计时内容
显示距离最近的一个假期（寒假或暑假）的倒计时
显示格式：XX天XX小时XX分钟
动态更新倒计时
假期时间设置
寒假时间：每年1月中旬至2月中旬
暑假时间：每年7月初至8月底
支持手动调整具体起止日期
界面设计
使用醒目的数字显示剩余天数
配以简单的动画效果
可选择温馨提示语（如"距离暑假还有..."）

2. 励志金句卡片
功能目标
点击“励志日历”首页菜单，进入新页面，通过精美的日历卡片形式展示励志金句，激励用户保持积极向上的心态。
具体需求
卡片样式
采用日历翻页的视觉效果
显示当前日期、星期
配以精美的背景图案
金句内容
预设100条精选励志金句
每天随机展示一条
分类包括：学习、运动、生活、梦想等
交互功能
支持左右滑动切换不同金句
点击可放大查看
长按可保存图片
分享功能
支持分享到朋友圈
分享给好友
保存到相册

3. 心情卡片
功能目标
点击“心情卡片”首页菜单，进入新页面，让用户通过记录每日心情和事件，生成个性化的心情卡片。
具体需求
卡片生成界面
日期选择（默认今天）
心情选择（开心、平静、难过、生气、兴奋）
文字输入框（限制100字）
卡片样式
提供3种基础背景：
暖色系（黄色、粉色等）
冷色系（蓝色、紫色等）
中性色系（白色、灰色等）
固定尺寸：750rpx宽度
卡片内容布局
顶部：日期显示
中间：心情表情
下方：文字内容
底部：简单装饰元素
基础功能
预览功能
一键保存
分享功能
历史记录查看

## 帮手通知系统
### 功能目标
通过短信通知机制，加强帮手对用户运动和游戏时间的监督作用，提高用户运动的积极性和游戏时间的管理效果。

### 具体需求
#### 帮手信息管理
- [ ] 帮手手机号码设置
  - 手机号码验证
  - 修改历史记录
- [ ] 帮手权限管理
  - 验证权限设置
  - 通知频率控制

#### 短信通知场景
- [ ] 运动验证通知
  - 运动记录提交时发送验证码
  - 验证结果反馈
- [ ] 监督提醒通知
  - 连续3天未运动提醒
  - 超额使用游戏时间警告
  - 成就解锁鼓励通知

## 数据分析功能
### 功能目标
为用户提供直观的运动和游戏数据分析，帮助用户更好地了解自己的运动和游戏习惯。

### 具体需求
#### 数据可视化
- [ ] 运动数据图表
  - 每日运动时长趋势
  - 运动类型分布
- [ ] 游戏时间分析
  - 使用时间分布
  - 兑换效率统计

#### 数据报告
- [ ] 周期性报告
  - 每周运动总结
  - 月度习惯分析
- [ ] 成就进度追踪
  - 目标完成度
  - 习惯养成进度

### 功能目标
增加用户之间的互动和激励，通过社交分享提高用户的运动积极性。

### 具体需求
#### 分享内容
- [ ] 运动成就分享
  - 里程碑达成
  - 特殊成就解锁
- [ ] 数据卡片分享
  - 每日运动记录
  - 周期性总结

#### 社交激励
- [ ] 点赞与评论
  - 好友互动
  - 鼓励留言
- [ ] 排行榜系统
  - 运动时长排名
  - 成就点数排名

### 1. 用户系统
- [ ] 用户注册与登录功能
  - 支持微信授权登录
  - 收集用户基本信息
  - 必须绑定帮手手机号码

### 2. 运动记录系统
- [ ] 运动时间记录

  - 运动类型选择（可扩展）
- [ ] 验证机制

  - 模拟发送验证码至帮手手机
  - 验证码验证功能

### 3. 游戏时间管理
- [ ] 时间兑换系统
  - 自动计算可兑换的游戏时间

- [ ] 游戏时间控制
  - 倒计时功能
  - 每30分钟强制休息提醒
  - 休息时间设置（预留）

### 4. 数据统计与展示
- [ ] 运动数据统计
  - 累计运动时长
  - 运动频次统计
- [ ] 游戏时间统计
  - 剩余可用时长
  - 使用记录

## 非功能需求

### 1. 性能需求
- 页面加载时间：<3秒
- 数据同步延迟：<1秒
- 验证码发送延迟：<5秒

### 2. 安全需求
- 用户数据加密存储
- 验证码有效期限制
- 防刷机制

### 3. 可用性需求
- 界面简洁直观
- 操作步骤最小化
- 适配不同尺寸设备

### 4. 可维护性需求
- 模块化设计
- 代码注释完善
- 错误日志记录

## 技术要求

### 1. 开发环境
- 微信开发者工具
- Node.js环境
- 云开发环境

### 2. 技术栈
- 前端：原生微信小程序
- 后端：微信云开发
- 数据库：云数据库

### 3. 第三方服务
- 短信服务（预留）
- 数据分析服务（预留）

## 项目限制
1. 仅支持微信小程序平台
2. 使用微信云开发，不部署独立服务器
3. MVP版本功能以基础需求为主

## 风险评估
1. 用户参与度不足
2. 验证流程可能被绕过
3. 游戏时间控制可能被规避

## 后续规划
1. 引入更多激励机制
2. 接入实际短信服务
3. 增加社交功能
4. 引入更多数据分析
